  • 1 - What happend when the user type something inside the input form ?

    When the user types something inside the input form, an event listener is added to the HTML form element on the page.

    It listens for the "submit" event to occur when the user submits the form by clicking a button or pressing enter.

    When the event occurs, the function prevents the default form submission behavior, which would cause the page to reload.

    Instead, it calls the createTaskItem function and passes it the current value of the taskInput field as a parameter.

    The purpose of this function is to allow the user to submit a new task to the task list. By preventing the default form submission behavior, the page can remain interactive and update dynamically as the user interacts with it.

  • 2 - What happend when the user clicks the Add button?

    When you click on the add button, the function createTaskItem() is called with the value of the taskInput element as the argument.

    The function creates a new li element and adds a span and a button element to it, sets their text content and attributes, and adds event listeners to the button and span elements.

    The function then adds the li element to the taskList element and resets the value of the taskInput element to an empty string.

    Additionally, the function changeBackgroundColor() is called, which changes the background color of the li element with class action-2 to green, and sets a timeout of 5 seconds to reset the background color to white if the input field is empty.

    The event listener on the taskInput element also changes the background color of the li element with class action-1 to green every time the input value changes, and resets the background color to white if the input field is empty.
